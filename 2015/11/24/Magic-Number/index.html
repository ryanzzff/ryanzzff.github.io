Magic Number

2038

My company has a web application that use cookie to store login information, which works well… until an device’s embedded system is upgraded.
The application logout the user every time when the app quits. After some investigations, we found it is related to the Year 2038 problem as the programmer have set the expiry date to now() + 99 year, which is larger than 2038 and the magic happened.

Year 2038 problem is a problem like the Y2K problem, as most 32-bit Unix-like os is using signed 32-bit integer to store time in unix-timestamp. Therefore we can only use this format to represent time until 19/1/2038, (2^31 - 1 = 2147483647 seconds after 1/1/1970). After that the integer will overflow and the world ends. :)
